CentOS 7.6 安装RocketMQ4.5.2(一)

发表于 2019-10-24 分类于 软件安装 Valine：

如何在CentOS7.6中安装RockeMQ

环境信息

硬件环境：阿里云 1CPU 1G内存 20G硬盘
软件环境：CentOS_7.6 JDK_1.8.231 Maven_3.5.2

安装准备

下载RocketMQ

于Github中下载RocketMQ源码：RocketMQ

上传RocketMQ

由于使用的是MacOS系统，这里使用 Transmit 这个软件进行文件上传,上传至/data目录

安装RocketMQ

1.使用# cd /data命令进入/data目录中

2.使用# unzip -n rocketmq-master.zip -d /usr/local/rocketmq命令，将安装包解压缩到/usr/local/rocketmq目录

如果提示unzip命令不可用，需先安装unzip组件

3.进入/usr/local/rocketmq目录，发现目录下多了个rocketmq-master文件夹，进入该文件夹

4.使用命令# mvn -Prelease-all -DskipTests clean install -U，进行打包

5.经过一定时间的等待，提示打包完成，这里耗时4分07秒

6.进入 /data/usr/local/rocketmq/rocketmq-master/distribution/target/rocketmq-4.5.2/rocketmq-4.5.2目录

配置启动RocketMQ

启动项目

进入目录

进入/usr/local/rocketmq/rocketmq-master/distribution/target/rocketmq-4.5.2/rocketmq-4.5.2/目录
命令：# cd /usr/local/rocketmq/rocketmq-master/distribution/target/rocketmq-4.5.2/rocketmq-4.5.2/

启动NameServer

命令# nohup sh bin/mqnamesrv &

发现，启动失败，查看启动日志，检查原因，命令：# tail -f nohup.out，发现是内存不足导致的。RocketMQ默认设置的启动内存较高，而笔者使用的是1G内存的服务器。

修改启动内存
修改bin目录下runserver.sh和runbroker.sh两个文件中关于启动内存的配置
runserver.sh:
修改值为JAVA_OPT="${JAVA_OPT} -server -Xms256m -Xmx256m -Xmn125m -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=320m"

runbroker.sh:
修改值为JAVA_OPT="${JAVA_OPT} -server -Xms256m -Xmx256m -Xmn125m"

重新启动NameServer# nohup sh bin/mqnamesrv &，此时没有退出，使用# tail -f nohup.out查看日志，出现The Name Server boot success. serializeType=JSON，表明NameServer启动成功

启动Broker

启动命令# nohup sh bin/mqbroker -n localhost:9876 &，并查看结果# tail -f nohup.out，当出现The broker[dev, 172.16.10.161:10911] boot success. serializeType=JSON and name server is localhost:9876时表明Broker启动成功

尾声

以上即为CentOS7.6安装RocketMQ4.5.2的详细过程
